1. Contextualização Extensionista
O sistema foi concebido para atender a uma demanda real de uma MEI de Manutenção de Computadores, focando na organização de processos internos e acompanhamento de demandas de clientes. O principal objetivo é transitar de um controle informal para um Sistema Gerenciador de Tarefas Online centralizado.

2. Metodologia de Gerência
O desenvolvimento adotou abordagens ágeis, com a organização de tarefas em ciclos curtos e acompanhamento visual via quadro Kanban (link disponível no README). A gestão contemplou:

Definição de escopo baseada nas dores do parceiro.
Distribuição de responsabilidades e controle de prazos.
Identificação de funcionalidades essenciais para o fluxo de manutenção.

3. Especificações das Funcionalidades (MVP)
Conforme os requisitos do Projeto Integrador III-B, o protótipo implementa:

Autenticação de Usuários: Interface de login para garantir acesso restrito aos dados de serviços e clientes.
Gestão de Ordens de Serviço: Criação de "projetos" de manutenção, permitindo definir prazos de entrega e descrição técnica do defeito.
Controle de Fluxo (Kanban): Colunas estruturadas em "A fazer", "Em andamento" e "Concluído", permitindo o rastreamento visual de cada máquina na bancada.
Monitoramento de Prazos: Integração com visualização de calendário para assegurar a pontualidade na entrega dos equipamentos consertados.
